## 2. أفضل 3 أوامر نصية (Prompts) تم استخدامها لبناء النسخة الاحترافية

### ⏹️ الأمر الأول: دمج هوية التصميم مع الـ SASS والـ Tailwind CSS
> **النص:** "قم بأخذ فكرة مشروع 'منصة أُفق لريادة الأعمال' وحولها إلى لوحة تحكم وتطبيق ويب فخم جداً ومناسب لتصوير فيديو شرح أكاديمي. حافظ على الهوية اللونية الفاخرة (الأخضر الملكي الداكن #1a4a2e والذهبي الفاخر #c9a84c) مع خطوط رويال مثل Amiri للعناوين و Tajawal للنصوص. استخدم Tailwind CSS لبناء واجهة بطاقات مهام ذكية تشبه الأنظمة السحابية المتقدمة."

### ⏹️ الأمر الثاني: صياغة الـ CRUD والمزامنة السحابية الفورية بـ Supabase
> **النص:** "برمج التطبيق بالكامل باستخدام vanilla JavaScript للربط المباشر مع Supabase Client SDK. قم بتفعيل واجهة لتسجيل الدخول وإنشاء الحسابات المشفرة (Supabase Auth)، واربط نماذج الإضافة والتعديل والحذف (Full CRUD) بجدول سحابي حقيقي يسمى `projects` بحيث يربط كل مشروع تلقائياً بالرقم الفريد للمستخدم المسجل `user_id` تحقيقاً لشروط أمان وخصوصية السجلات."

### ⏹️ الأمر الثالث: بناء خط تتبع المراحل الخمس التفاعلي (Pipeline Stepper)
> **النص:** "اكتب دالة جافا سكريبت ذكية تقوم بقراءة حقل المرحلة `current_stage` من قاعدة البيانات لكل مشروع وتوليد خط تتبع رسومي تفاعلي (Stepper) من المرحلة 1 إلى 5 داخل بطاقة كل مشروع، ليظهر تقدم صاحب المشروع في تأهيل فكرته الاستثمارية بشكل فني رائع ومناسب جداً للشرح المباشر بالفيديو."

## 3. وصف نموذج البيانات لجدول السحابة (Data Model Schema)
لتشغيل المنصة بنجاح حقيقي أثناء تصوير الفيديو وتقييم المساق، تم تصميم الجدول في **Supabase** تحت اسم **`projects`** بالبنية الفنية التالية:

| اسم الحقل البرمجي | نوع الحقل في السحابة | الشرح والاستخدام الفني |
| :--- | :--- | :--- |
| `id` | UUID (Primary Key) | المفتاح الرئيسي المولد تلقائياً فريداً لكل طلب |
| `created_at` | Timestamptz | تاريخ ووقت تقديم المقترح تلقائياً |
| `user_id` | UUID (References auth.users) | حقل التحقق من هوية وأمان صاحب الحساب المسجل |
| `title` | Text | اسم أو عنوان المشروع الريادي المقدم |
| `category` | Text | قطاع وتوجه الفكرة الاستثمارية (تقنية، تجارة...) |
| `current_stage` | Text / Number | مرحلة التأهيل الحالية وتأخذ قيم من (1 إلى 5) |
| `status` | Text | حالة الطلب التشغيلية (قيد المراجعة، مقبول، يتطلب تعديل) |
| `description` | Text | شرح ونطاق العمل المفصل للفكرة والجمهور المستهدف |
